# Contribuer au projet WordPress Playground

WordPress Playground est un projet open-source qui accueille des contributeurs et contributrices de tous horizons, du code au design, de la documentation au triage.

<!--

-   Code? See the [developer section](/contributing/code).

-   Ideas, designs, or anything else? Open a [GitHub discussion](https://github.com/WordPress/wordpress-playground/discussions), and let's talk!

## Comment puis-je contribuer ?

-   Code ? Consultez la [section développeur](/contributing/code).
-   Documentation ? Consultez la [section documentation](/contributing/documentation).
-   Signaler des bugs ? Ouvrez une [nouvelle issue](https://github.com/WordPress/wordpress-playground/issues/new) dans le dépôt GitHub principal, ou dans [Playground Tools](https://github.com/WordPress/playground-tools/issues/new).
-   Idées, designs ou autre chose ? Ouvrez une [discussion GitHub](https://github.com/WordPress/wordpress-playground/discussions), et discutons !

## Directives

-   Comme pour tous les projets WordPress, nous voulons assurer un environnement accueillant et respectueux pour tous. Veuillez lire le [Code de conduite](https://make.wordpress.org/handbook/community-code-of-conduct/) de notre communauté pour en savoir plus.
-   Les contributeurs de code doivent examiner les [principes de codage](/contributing/coding-standards).
-   Vous conservez les droits d'auteur sur toute contribution que vous faites. En soumettant une Pull Request, vous acceptez de libérer ce code sous la [Licence WordPress Playground](https://github.com/WordPress/wordpress-playground?tab=GPL-2.0-1-ov-file#readme).

## Triage des issues

Vous voulez aider à trier les issues ouvertes et résoudre les bugs potentiels ? Voici comment faire :

1. Examinez la [liste des issues ouvertes](https://github.com/WordPress/wordpress-playground/issues?q=is%3Aopen+is%3Aissue) et trouvez celles avec lesquelles vous pouvez aider. Il en va de même pour le [dépôt Playground Tools](https://github.com/WordPress/playground-tools/issues?q=is%3Aopen+is%3Aissue).
2. Lisez la description et les commentaires.
3. Si c'est un bug que vous pouvez reproduire, ajoutez un commentaire descriptif ou une correction potentielle.
4. Sinon, ajoutez un commentaire avec toute information supplémentaire qui pourrait être utile.

## Une note sur la contribution et la licence GPL

WordPress Playground et le projet WordPress sont fortement ancrés dans le logiciel libre et open source. Plus précisément, WordPress Playground est licencié sous GPLv2 (ou ultérieur) de la [Free Software Foundation](https://www.fsf.org/). Vous pouvez [lire le texte de la licence ici](https://github.com/WordPress/wordpress-playground/blob/trunk/LICENSE) et si cela semble accablant, WordPress.org a un [Primer GPL convivial](https://make.wordpress.org/community/handbook/wordcamp-organizer/planning-details/gpl-primer/).

En tant que tel, veuillez être conscient des implications sous lesquelles vos contributions tomberont :

-   Lorsque vous contribuez, vous acceptez de licencier vos contributions sous la licence GPLv2 (ou ultérieure)
-   La licence GPL a de fortes dispositions de copyleft qui garantissent que toutes les œuvres dérivées restent open-source et sous les mêmes termes de licence, promouvant ainsi un environnement de développement collaboratif.
-   La licence GPL encourage la contribution de tout changement, correction de bug ou nouvelle fonctionnalité au code source original.
-   La licence GPL garantit que le projet reste libre et open-source, non seulement en termes de coût mais aussi en ce qui concerne la liberté d'utiliser, de modifier et de distribuer le logiciel.
